<h2><a id="user-content-get-container-image" class="anchor" aria-hidden="true" href="#get-container-image"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Get container image</h2>
<p>To get the latest version of the container image run:</p>
<div class="highlight highlight-source-shell"><pre>docker pull akashin/coursera-aml-nlp</pre></div>
<p>It containes Ubuntu 16.04 Linux distirbutive and all dependencies that you need for our course. The downloaded image takes approximately 2.3GB.</p>
<p><strong>Note:</strong> If you are getting an error "Got permission denied while trying to connect to the Docker daemon socket...", you need to add current user to the docker group:</p>
<div class="highlight highlight-source-shell"><pre>sudo usermod -a -G docker <span class="pl-smi">$USER</span>
sudo service docker restart</pre></div>
<p>Then you need to logout and login to the system again (disconnect and connect to your AWS instance if you are setting up a docker on it).</p>
<h2><a id="user-content-run-container-for-the-first-time" class="anchor" aria-hidden="true" href="#run-container-for-the-first-time"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Run container for the first time</h2>
<p>Now you can start new container from this image with:</p>
<div class="highlight highlight-source-shell"><pre>docker run -it -p 8080:8080 --name coursera-aml-nlp akashin/coursera-aml-nlp</pre></div>
<p>This will start the Ubuntu instance and give you an access to its command line. You can type <code>run_notebook</code> to launch IPython notebook server.</p>
<p>You may find it useful to mount a directory from your local machine within the container using <code>-v</code> option.</p>
<p>For Linux and OSX, the following command should work:</p>
<div class="highlight highlight-source-shell"><pre>docker run -it -p 8080:8080 --name coursera-aml-nlp -v <span class="pl-smi">$PWD</span>:/root/coursera akashin/coursera-aml-nlp</pre></div>
<p>This will use shell alias <code>$PWD</code> to mount current directory to the folder <code>/root/coursera</code> in the container. Alternatively, you can mount arbitrary directory by replacing <code>$PWD</code> with a custom path.</p>
<p>For Windows, there are some extra <a href="https://rominirani.com/docker-on-windows-mounting-host-directories-d96f3f056a2c" rel="nofollow">steps</a> involved, and the launch command looks like</p>
<div class="highlight highlight-source-shell"><pre>docker run -it -p 8080:8080 --name coursera-aml-nlp --user root -v /c/Users/<span class="pl-smi">$YOUR_USERNAME</span>:/root/coursera akashin/coursera-aml-nlp</pre></div>
<p>Where <code>/c/Users/$YOUR_USERNAME</code> is the path to your user's home folder.</p>
<p>If you're using Docker Toolbox on Windows, the command given above might not work because of the additional VirtualBox layer involved. Instead, we recommend you to follow the guidance in <a href="http://blog.shahinrostami.com/2017/11/docker-toolbox-windows-7-shared-volumes/" rel="nofollow">http://blog.shahinrostami.com/2017/11/docker-toolbox-windows-7-shared-volumes/</a>.</p>
<h2><a id="user-content-stop-and-resume-container" class="anchor" aria-hidden="true" href="#stop-and-resume-container"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Stop and resume container</h2>
<p>To stop the container use:</p>
<div class="highlight highlight-source-shell"><pre>docker stop coursera-aml-nlp</pre></div>
<p>All the changes that were made within container will be saved.</p>
<p>To resume the stopped container use:</p>
<div class="highlight highlight-source-shell"><pre>docker start -i coursera-aml-nlp</pre></div>
<h2><a id="user-content-other-operations-on-the-container" class="anchor" aria-hidden="true" href="#other-operations-on-the-container"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Other operations on the container</h2>
<p>There are many other operations that you can perform on the container, to show all of them:</p>
<div class="highlight highlight-source-shell"><pre>docker container</pre></div>
<p>Some particularly useful would be <strong>showing a list of containers</strong> and <strong>removing container</strong>.</p>
<p>To show currently running and stopped containers with their status:</p>
<div class="highlight highlight-source-shell"><pre>docker ps -a</pre></div>
<p>To connect to a Bash shell in the already running container with name <code>coursera-aml-nlp</code> run:</p>
<pre><code>docker exec -it coursera-aml-nlp bash
</code></pre>
<p>This will drop you into the standard Linux Bash shell that supports common commands like <code>ls</code>, <code>wget</code> or <code>python3</code>.</p>
<p>To remove the container and all data associated with it:</p>
<div class="highlight highlight-source-shell"><pre>docker rm coursera-aml-nlp</pre></div>
<p>Note, that this will remove all the internal data of the container (e.g. installed packages), but all the data written inside of your local mounted folder (<code>-v</code> option) will not be affected.</p>
<h2><a id="user-content-install-more-packages" class="anchor" aria-hidden="true" href="#install-more-packages"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Install more packages</h2>

<p>If you try to open "<a href="http://localhost:8080" rel="nofollow">http://localhost:8080</a>" or "<a href="http://127.0.0.1:8080" rel="nofollow">http://127.0.0.1:8080</a>" in your browser, when <code>run_notebook</code> command is started, and you can't access your notebooks, here are some advices:</p>
<ul>
<li>If you're using Docker Toolbox on Windows, try accessing "<a href="http://192.168.99.100:8080" rel="nofollow">http://192.168.99.100:8080</a>" instead. If this doesn't work, follow the instructions <a href="https://docs.docker.com/docker-for-windows/troubleshoot/#limitations-of-windows-containers-for-localhost-and-published-ports" rel="nofollow">on official Docker docs</a> and on <a href="https://stackoverflow.com/questions/42866013/docker-toolbox-localhost-not-working" rel="nofollow">Stackoverflow</a>.</li>
<li>Make sure that you're running container with <code>-p</code> flag as described <a href="#run-container-for-the-first-time">here</a> and that the output of <code>docker ps</code> contains a message like this:</li>
</ul>
<pre><code>CONTAINER ID        IMAGE                      COMMAND             CREATED                  STATUS              PORTS               NAMES
e5b7bcd85a1b        akashin/coursera-aml-nlp   "/bin/bash"         Less than a second ago   Up 2 seconds        8080/tcp            peaceful_lamarr
</code></pre>
<p>If the part about <code>PORTS</code> differs, remove the current container following <a href="#other-operations-on-the-container">instructions</a> and start it again.</p>
<ul>
<li>Make sure that browser proxy settings don't interfere with accessing local web sites.</li>
</ul>
<h3><a id="user-content-how-do-i-load-data-into-docker-container" class="anchor" aria-hidden="true" href="#how-do-i-load-data-into-docker-container"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>How do I load data into Docker container?</h3>
<p>To access the data in the container, we recommend to use <code>-v</code> flag described <a href="#run-container-for-the-first-time">here</a> to mount a local directory from your computer into the container filesystem. For more details read <a href="https://docs.docker.com/storage/volumes/" rel="nofollow">Docker documentation</a>.</p>
<p>Alternatively, you can download data using Jupyter "Upload" button or <code>wget</code> command in the <a href="#other-operations-on-the-container">Bash shell</a> of the container.</p>
<h3><a id="user-content-cant-run-run_notebook-or-starspace-command" class="anchor" aria-hidden="true" href="#cant-run-run_notebook-or-starspace-command"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Can't run <code>run_notebook</code> or <code>starspace</code> command</h3>
<p>Make sure that you're executing it in the context of the Docker container as described <a href="#run-container-for-the-first-time">here</a>.</p>
<h3><a id="user-content-name-is-already-in-use-by-container-when-trying-to-run-the-container" class="anchor" aria-hidden="true" href="#name-is-already-in-use-by-container-when-trying-to-run-the-container"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>"Name is already in use by container" when trying to run the container</h3>
<p>This means that the container with this name is already created. You can connect to this container or remove it by following <a href="#other-operations-on-the-container">instructions</a>.</p>
<h3><a id="user-content-starspacejupyter-notebook-crashes-in-docker" class="anchor" aria-hidden="true" href="#starspacejupyter-notebook-crashes-in-docker"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>StarSpace/Jupyter notebook crashes in Docker</h3>
<p>This usually happens due to low default 2GB memory limit on Windows and OSX. Follow this <a href="#change-ram-limits-of-the-container">instructions</a> to fix this.</p>
<h3><a id="user-content-this-computer-doesnt-have-vt-xamd-v-enabled-when-trying-to-run-the-container" class="anchor" aria-hidden="true" href="#this-computer-doesnt-have-vt-xamd-v-enabled-when-trying-to-run-the-container"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>"This computer doesn't have VT-X/AMD-v enabled", when trying to run the container</h3>
<p>This usually happens if you're using Docker Toolbox that needs Virtual Box support - hence the need for the hardware virtualization that can be enabled in BIOS.
Try to turn on the VT-X support in BIOS as described in <a href="https://blogs.technet.microsoft.com/canitpro/2015/09/08/step-by-step-enabling-hyper-v-for-use-on-windows-10/" rel="nofollow">Microsoft documentation</a> or on <a href="https://github.com/docker/machine/issues/4271">GitHub</a>.</p>
